# References

Frameworks, checklists, and reusable notes for this path. Things you reach for more than once: a listing checklist, a script you reuse, a vendor list, a how-to.

These are different from `decisions/`. A decision is a call you made on a date. A reference is a thing you keep using. References can be updated as they improve (on Claude, by you, since the connector cannot edit).

Save one with "save this to my references" and the save-to-os skill files it here.
