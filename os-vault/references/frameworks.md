# Frameworks

The OS is built on two frameworks from Nate Herk's AIS-OS (github.com/nateherkai/AIS-OS). Credit to him. They are here as the thinking tools the weekly skills use, adapted for a browser and Google Drive setup. "The Three Ms of AI" is Nate Herk's trademark; the framework content is used under his MIT license.

## The Four Cs (the architecture)

A way to check the OS is complete. The audit walks these in order.

1. **Context.** What does Claude know about you and your work? Lives in `context/`. Thin context means generic answers.
2. **Connections.** What tools can Claude reach? Lives in `connections.md`. A tool not connected is a blind spot.
3. **Capabilities.** What can Claude actually do for you? The Skills, and the saved prompts you reuse.
4. **Cadence.** How often does the OS run and stay current? The weekly audit, the daily rundown, the decision log.

A gap in any one is where the system leaks.

## The Three Ms (the operator shift)

A way to decide what to hand off next.

1. **Mindset.** Stop doing the task yourself first. Ask whether the OS could.
2. **Method.** Write down how you do the task, in plain steps. A method written down is a method that can be handed off.
3. **Machine.** Turn the method into a Skill or a saved prompt so it runs the same way every time.

The weekly level-up pass takes one thing you did by hand this week and moves it from Mindset to Machine.
