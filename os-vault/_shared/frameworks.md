# Frameworks

The OS is built on two frameworks from Nate Herk's AIS-OS (github.com/nateherkai/AIS-OS). Credit to him. They are here as the thinking tools the weekly audit and level-up use, adapted for a browser and Google Drive setup.

## The Four Cs (the architecture)

A way to check the OS is complete. The audit walks these in order.

1. **Context.** What does the AI know about you and each area? Lives in each path's `overview.md`. Thin context means generic answers.
2. **Connections.** What tools can the AI reach? Lives in `connections.md`. A tool not connected is a blind spot.
3. **Capabilities.** What can the AI actually do for you? The skills, and the saved prompts you reuse.
4. **Cadence.** How often does the OS run and stay current? The weekly audit, the inbox triage, the decisions record.

A gap in any one is where the system leaks.

## The Three Ms (the operator shift)

A way to decide what to hand off next.

1. **Mindset.** Stop doing the task yourself first. Ask whether the OS could.
2. **Method.** Write down how you do the task, in plain steps. A method written down is a method that can be handed off.
3. **Machine.** Turn the method into a skill so it runs the same way every time.

The weekly level-up takes one thing you did by hand this week and moves it from Mindset to Machine.

## Capture and decay (what keeps it from rotting)

The Four Cs keep the OS complete. These two keep it honest over time.

- **Capture.** A thought you do not write down is lost. The `inbox/` is one line, fast, no filing. The weekly audit sorts it into the right path.
- **Decay.** An OS that only grows goes stale. Nothing is deleted, but old truth gets superseded by a new dated note, and dead material moves to `_archive/`. The audit flags what no longer matches reality.
