# Operating instructions

This is the brief your AI reads to operate. It works the same in Claude, Gemini, or ChatGPT. The depth lives in the rest of this folder. This file points there.

**Read `me.md` first.** That is who I am and how I want you to work, my voice and the lines you do not cross. It overrides anything here about how to work. This file is how the OS is wired.

## What this is

My AI operating system. It lives in this Google Drive folder so any AI I use can read it. It is split into paths (`business/`, `personal/`, `work/`), and only the ones I use are filled in. Each path has a living `overview.md`, a `references/` folder, and an append-only `decisions/`.

## How to access it

On any task, read what is relevant before answering:

- `me.md` — who I am and how to work with me. Read first.
- The `overview.md` of the path the task is about. Current state, people, process.
- `_shared/connections.md` — the tools you can reach and what each is for.
- The path's `decisions/` — decisions and changes already made. Check it before assuming.
- `_shared/frameworks.md` — the thinking the OS runs on.
- `_shared/skills/INDEX.md` — the skills you have and what each does.

## How to work

- Do the task. Do not narrate how you would do it. If one fact is missing, ask one specific question and stop.
- Check the right source before answering: the path overview for context, `connections.md` for which tool holds what, the path's `decisions/` before assuming a call has not been made.
- When I ask you to save, log, or record something, write a new dated note in the right path's `decisions/`. Never overwrite an existing note.
- When something looks off, say so without being asked.

## Saving and platform

How saves work depends on the platform. See `_shared/platforms.md`. Short version: on Claude every save is a new dated file because the connector cannot edit. On Gemini and ChatGPT you can keep the rollup current in place, but history still goes to `decisions/` as new notes. Never lose what was true before.
