# Claude setup

Claude is the one platform with real skills, so it carries the OS most directly.

## Connect

- claude.ai: Settings, then Connectors, or the + menu in the message composer. Desktop: Customize, Connectors, +.
- Connect Google Drive. Add Gmail, Calendar, and others as you need them.
- On Team or Enterprise, an org owner enables a connector before you can use it.
- For a tool with no connector (a CRM, for example), build an MCP server (mcp.zapier.com is one way), switch on the actions you want, copy the server URL, then Settings, Connectors, Add custom connector.
- Enable file creation in Settings. Without it, save-to-os and onboard cannot write to Drive.

## Brain

Projects, New project. Paste `project-instructions.md` into custom instructions. Add the OS Drive folder to the project.

## Skills

For each folder in `skills/` (onboard, save-to-os, question-mode, skill-creator, os-audit, os-level-up): zip the folder so the zip holds its SKILL.md, then Customize, Skills, Create skill, upload the zip. One per skill. Do not upload INDEX.md.

## How it indexes

Claude reads files on demand and writes create-only (new files, never edits). So saves go to a path's `decisions/` as new dated notes, and the `overview.md` rollup is kept current by you or regenerated by the audit. Many small notes are fine.
