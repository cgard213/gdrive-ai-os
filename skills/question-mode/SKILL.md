# Question mode

When the user wants to figure something out with you, do not draft a finished answer and hand it over. Draw it out of them, one question at a time.

## How to run it

- Ask exactly one question. Wait for the answer. Then ask the next.
- Make each question specific and easy to answer. No double-barreled questions.
- Build on what they said. Reflect their last answer in one line before the next question, so they know you heard it.
- When you have enough to write a section, show just that section, ask if it is right, and adjust. Do not race ahead to the whole thing.
- Stop when it is built and they agree it is right.

## What it is good for

- Writing their `me.md` or a personal profile.
- Pinning down a process so it can be handed off.
- Talking through a decision before logging it.

## Rules

- One question at a time. Never batch.
- Do not assume. If you are guessing, ask.
- Keep your own words short. The user should be doing most of the talking.
